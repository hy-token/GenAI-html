第1章: Godotとは何が違うのか

1.1 完全オープンソースの意味とコスト優位性

Godotの最大の特徴は、完全なオープンソースソフトウェアであることです。これは、ゲーム開発業界の従来の常識を覆す革新的な点です。

主要エンジンのライセンス比較

エンジン 無料版の制限 有料版の料金 ロイヤリティ
Unity 年間売上$20万まで $1,800～/年 なし
Unreal Engine 完全無料 Epic Games Store配信時のみ 売上の5%
Godot 一切なし 無料 なし

Godotでは以下のようなビジネス上の利点があります：

  • ライセンス費用が0円：企業規模に関係なく完全無料で使用可能
  • ロイヤリティフリー：収益に関係なく追加費用なし
  • 商用利用制限なし：小規模から大規模まで自由に利用可能
  • ソースコード公開：必要に応じてエンジン自体をカスタマイズ可能

1.3 ノードベース設計の革新性

Godotの最も革新的な設計思想は、すべてがノードで構築される点です。これは従来のGameObject（Unity）やActor（Unreal）とは根本的に異なります。

# Godotのノード構成例 ┌─ Node2D（キャラクター） │ ├─ Sprite2D（見た目） │ ├─ CollisionShape2D（当たり判定） │ ├─ AudioStreamPlayer2D（効果音） │ └─ [Script] character.gd

ノードベース設計の利点

  • 直感的な階層構造：親子関係が視覚的に分かりやすい
  • 再利用性：ノードをそのままシーンとして保存・再利用可能
  • コンポーネント指向：機能ごとにノードを組み合わせて構築
  • デバッグの容易さ：ノード単位で動作確認が可能

1.4 GDScriptの独自性（Python/C#/C++との違い）

GDScriptは、Godot専用に設計されたスクリプト言語です。Pythonに似た文法を持ちながら、ゲーム開発に特化した機能を持っています。

# GDScript例：基本的なキャラクター移動 extends CharacterBody2D var speed = 300.0 func _process(delta): var direction = Input.get_vector("ui_left", "ui_right", "ui_up", "ui_down") velocity = direction * speed move_and_slide()

GDScriptの特徴

特徴 説明 他言語との違い
ゲーム専用設計 Godotエディタと完全統合 別ファイルの設定が不要
型システム 動的型付けと静的型付けのハイブリッド Pythonより安全、C#より簡潔
パフォーマンス JITコンパイル対応 Pythonより高速
学習曲線 Pythonライクな文法 初心者に優しい

1.5 ライセンス問題からの解放

Godotは完全オープンソース（MIT License）のため、以下のような自由度があります：

Godotの自由度

  • ソースコードの修正・再配布が可能
  • 独自のエンジン改造が許可
  • 商用利用時の報告義務なし
  • クレジット表示の任意性
  • プラットフォーム制限なし

他エンジンとの比較

  • Unity：Unity Hubアカウント必須、ライセンス管理が複雑
  • Unreal Engine：Epic Gamesアカウント必須、ソースコード利用に制限
  • Godot：アカウント不要、完全にオフラインで使用可能

1.6 クロスプラットフォーム対応の実際

Godotは、ワンクリックで複数プラットフォームに対応できる優れたエクスポート機能を持っています。

プラットフォーム 対応状況 追加費用 特別な設定
Windows ✓ 完全対応 無料 不要
macOS ✓ 完全対応 無料 署名が必要
Linux ✓ 完全対応 無料 不要
Android ✓ 完全対応 無料 Android Studio必要
iOS ✓ 完全対応 Apple Developer登録必要 Xcode必要
Web (HTML5) ✓ 完全対応 無料 WebASM対応
